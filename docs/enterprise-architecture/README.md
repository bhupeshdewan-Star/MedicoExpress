## Authority Rules

- The Project Constitution contains stable product principles and requires an explicit decision record to change.
- Specifications evolve through reviewed changes recorded in the Decision Log.
- The Delivery Board is updated daily and is the operational source of truth.
- Existing ClinCommand code and documents are evidence and candidate assets, not architectural authority.
- Every regulated activity must consult an effective SOP and the required skill definitions before execution.
- No AI output becomes an approved organizational record without the required human validation.

## Definition of Living Memory

Living memory means:

- decisions are written before they disappear into conversation;
- assumptions are visibly marked and reviewed;
- every specification has a status and owner;
- changes remain traceable;
- delivery work links back to an architectural requirement;
- lessons from implementation update the relevant specification.

## Document Status Vocabulary

| Status | Meaning |
|---|---|
| Draft | Under active development; not yet binding |
| Proposed | Ready for stakeholder review |
| Accepted | Approved as an implementation constraint |
| Superseded | Replaced by a newer documented decision |
| Retired | No longer applicable |
