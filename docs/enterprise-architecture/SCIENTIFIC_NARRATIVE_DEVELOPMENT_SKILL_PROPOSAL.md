# Scientific Narrative Development Skill Proposal

**Document status:** Draft for owner review  
**Version:** 0.1  
**Related activity:** Scientific Narrative Development  
**Primary citation style:** Vancouver style  

## 1. Working Title

Scientific Narrative Development and Validation

## 2. Purpose

This skill helps ClinCommand OS produce a compelling scientific narrative that is:

- evidence-led;
- clinically meaningful;
- easy to follow;
- balanced;
- source-traceable;
- suitable for internal or scientific use.

The skill is designed to make the scientific story feel interesting without losing precision.

## 3. Positioning

This skill is not a product monograph skill, not a slide-training skill, and not a generic summary writer.

It sits between:

- the approved source pack;
- the evidence hierarchy;
- the final scientific narrative package.

## 4. Source Hierarchy

The skill must resolve sources in the following order:

1. owner-approved SOPs and skill files;
2. controlled repository assets;
3. approved project memory and architecture notes;
4. peer-reviewed articles and official guideline sources;
5. external skill registries only for skill-improvement and pattern learning.

When skill discovery or training is needed, the review order should include:

- skills.sh
- Agent Skills Me
- SkillsMP
- SkillsDirectory
- Smithery.ai

Only a small shortlist of the best matching patterns should be adopted, and only after review.

## 5. Required Human / AI Roles

The subagent bundle for this skill should include:

| Role | Function |
|---|---|
| Planner bot | Locks scope, audience, and story objective |
| Source bot | Collects literature, guidelines, and approved source packs |
| Story bot | Builds the scientific narrative spine |
| Citation bot | Formats and checks Vancouver references |
| QA bot | Checks logic, readability, and completeness |
| Compliance bot | Checks label, jurisdiction, and fair-balance boundaries |
| Memory bot | Stores what improved the draft and what failed |

## 6. Mandatory Inputs

The skill should require:

- topic or molecule;
- audience;
- geography;
- desired depth;
- intended use;
- approved evidence pack or source list;
- label / scope boundary;
- output length or page target;
- whether visuals or tables are needed;
- whether downstream reuse is expected.

## 7. Required Capabilities

The skill must be able to:

1. detect the central scientific question;
2. select the most relevant evidence;
3. build a narrative arc rather than a flat summary;
4. keep the tone scientific and interesting;
5. avoid duplicate ideas and repeated graphics unless repetition is justified;
6. separate facts, interpretation, and limitations;
7. maintain Vancouver-style citation discipline;
8. produce a reference map for auditability;
9. support two-pass drafting and refinement;
10. generate memory notes for self-improvement.

## 8. Output Architecture

The skill should produce:

1. narrative objective
2. audience and use context
3. story spine
4. evidence blocks
5. interpretation layer
6. limitation / caution layer
7. practical implication layer
8. Vancouver reference list
9. claim traceability notes
10. revision triggers

## 9. Writing Standard

The narrative should:

- open with a clinical reason to care;
- keep paragraphs readable and purposeful;
- use transitions that move the story forward;
- avoid dull encyclopedic repetition;
- avoid jargon when simpler scientific language works;
- avoid rhetorical clutter and hidden instructions;
- keep citation numbering consistent throughout.

## 10. Controls

1. No unsupported claim.
2. No invisible source chain.
3. No citation style other than Vancouver for scientific content.
4. No promotional inflation.
5. No repeated visual or conceptual asset unless it clearly improves understanding.
6. No invented data or invented mechanism language.
7. No unsafe extrapolation from one population to another.

## 11. Quality Thresholds

The output is acceptable only if it demonstrates:

- accurate evidence hierarchy;
- coherent narrative flow;
- full citation traceability;
- balanced interpretation;
- clear practical relevance;
- safe reuse potential.

## 12. Self-Improvement Loop

After every attempt, the AI subagents must record:

- which source types were strongest;
- which opening hooks worked best;
- where the narrative became too flat;
- where repetition appeared;
- which citations or evidence blocks were most persuasive;
- what should be changed in the next draft.

That memory should be used to improve the next narrative, not to overwrite the evidence hierarchy.

## 13. Validation

Before release, the skill output must pass:

1. source traceability check;
2. Vancouver citation check;
3. story-flow check;
4. balance check;
5. terminology check;
6. duplication check;
7. compliance check;
8. human review.
