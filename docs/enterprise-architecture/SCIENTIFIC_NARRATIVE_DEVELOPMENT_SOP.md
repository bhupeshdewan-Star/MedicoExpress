# SOP for Scientific Narrative Development

**Document status:** Draft for owner review  
**Version:** 0.1  
**Activity:** Scientific Narrative Development  
**Owning domain:** Medical Affairs / Cross-Activity Medical Writing  
**Primary citation style:** Vancouver style  

## 1. Purpose

This SOP defines how ClinCommand OS prepares a scientific narrative that tells a clear, evidence-led story around a molecule, therapeutic area, product class, disease state, or development question.

The objective is to produce a narrative that is scientifically strong, readable, balanced, and suitable for internal review, scientific communication, and downstream reuse where allowed.

## 2. Scope

This SOP applies to:

- product-linked scientific narratives;
- disease-state narratives;
- therapeutic-area narratives;
- mechanism-to-clinical storyboards;
- evidence summaries intended to support scientific discussion;
- narrative content that may later feed monographs, slide decks, briefs, FAQs, or medical-information assets.

This SOP does not replace a product monograph SOP, a training-slide SOP, a publication SOP, or a clinical summary SOP.

## 3. Reference Style

All scientific claims, quotations, clinical statements, and comparative assertions must be referenced in **Vancouver style**.

Required reference behavior:

1. Use numbered in-text citations in the order they first appear.
2. Keep the running text clean and avoid URL-heavy citations.
3. Build a numbered reference list at the end.
4. Prefer peer-reviewed literature, guidelines, regulatory documents, trial registries, and approved local product information.
5. Use PubMed / PubMed Central / guideline sources where possible before secondary sources.

## 4. Responsibilities

### Lead medical writer or lead agent

- define the narrative objective;
- lock the intended audience;
- identify the central scientific tension;
- ensure the story stays within the approved evidence boundary;
- coordinate the source, draft, QA, compliance, and memory subagents.

### Source subagent

- retrieve and triage literature and other approved source packs;
- distinguish primary evidence from commentary;
- capture exact citation details for Vancouver formatting;
- flag missing, weak, or conflicting evidence.

### Narrative subagent

- convert evidence into a clear scientific story;
- choose the best narrative structure;
- keep the tone scientific, interesting, and restrained;
- avoid promotional inflation, unsupported superiority, or emotional overreach.

### QA subagent

- check logic, flow, terminology, and completeness;
- confirm every key claim has a citation;
- confirm that the running text does not repeat the same visual or conceptual device unnecessarily;
- confirm that the output does not read like a generic summary.

### Compliance subagent

- check label boundaries, fair balance, and jurisdictional restrictions;
- confirm that unsupported off-label claims are absent or explicitly marked as exploratory;
- check that references and claims are reviewable.

### Memory subagent

- capture lessons learned;
- store durable narrative patterns;
- record source patterns that consistently produce stronger outputs;
- track recurring weaknesses for retraining.

## 5. Required Inputs

The request must provide or allow discovery of:

- molecule, brand, disease state, or topic;
- target audience;
- jurisdiction or market;
- intended use;
- known label or source boundary;
- available approved source pack;
- mandatory claims or exclusions;
- desired length or depth;
- whether the output is for internal, scientific, or HCP-facing use.

## 6. Required Procedure

### Step 1: Intake and scope lock

- confirm the topic and audience;
- define the purpose of the narrative;
- identify the output format;
- list the likely source classes;
- record risks and missing inputs.

### Step 2: Source resolution

Consult sources in this order:

1. owner-supplied SOPs and skill files;
2. controlled repository materials;
3. approved project memory and architecture notes;
4. peer-reviewed publications and official guidance;
5. external skill registries only for skill-improvement research when required.

### Step 3: Evidence triage

- separate core evidence from supporting evidence;
- rank evidence by strength and relevance;
- remove weak, duplicated, outdated, or non-authoritative sources;
- record exact reference details for later Vancouver formatting.

### Step 4: Narrative architecture

The narrative should usually include:

- opening clinical tension;
- why the topic matters;
- what the evidence shows;
- where the evidence is limited;
- practical interpretation;
- balanced conclusion.

The narrative may be built as:

- problem -> evidence -> interpretation -> implication;
- mechanism -> proof -> clinical relevance -> caution;
- disease burden -> gap -> product or class role -> limitations.

### Step 5: Draft development

- write in a precise scientific tone;
- use short, readable paragraphs;
- use subheadings where helpful;
- keep claims proportionate to evidence;
- insert numbered citations at first mention;
- avoid repeating the same example, image, or phraseology across sections unless the repetition has a clear pedagogic purpose.

### Step 6: Scientific QA

Check for:

- missing references;
- unsupported claims;
- overstatement;
- duplicated visuals or repeated narrative devices;
- poor flow or abrupt transitions;
- hidden instruction language;
- source mismatch;
- reference style errors.

### Step 7: Compliance QA

Confirm:

- on-label or in-scope status;
- fair balance;
- privacy and consent where human material is used;
- jurisdictional restrictions;
- no implied superiority without support;
- no hidden promotional inflation.

### Step 8: Refinement pass

Perform one structured refinement after QA:

- tighten claims;
- improve readability;
- strengthen the opening hook;
- reduce clutter;
- update or replace weak evidence;
- normalize the Vancouver reference list.

### Step 9: Memory update

The memory subagent must record:

- which sources were most useful;
- which narrative structure worked best;
- which errors recurred;
- what should be carried forward next time;
- what should not be repeated.

## 7. Required Output Structure

Each scientific narrative should contain:

1. Title
2. Purpose
3. Audience and use case
4. Story spine
5. Scientific narrative body
6. Key evidence blocks
7. Limitations and cautions
8. Practical implications
9. Vancouver reference list
10. Revision notes

## 8. Controls

- No unreferenced medical claim.
- No generic filler paragraphs.
- No invented statistics.
- No copied editorial tone that disguises promotion.
- No unsupported superiority claim.
- No citation style drift away from Vancouver.
- No reuse of the same visual or graphic within a single deck or package unless the repetition is deliberate and justified.

## 9. Revision Triggers

This SOP must be reviewed when:

- new evidence materially changes the story;
- the label changes;
- a new safety issue appears;
- a new jurisdiction is added;
- the writing pattern repeatedly underperforms QA;
- the reference style or source hierarchy changes.

## 10. Training Requirement

Every AI subagent working under this SOP must complete at least one supervised draft, one QA cycle, and one refinement cycle before the output is considered mature.

The subagent must learn from each cycle by updating:

- preferred source ranking;
- evidence triage habits;
- narrative structure selection;
- citation discipline;
- visual or structural repetition control.
