# ADR-007: SaaS Billing Architecture

## Context
Transitioning to a commercial SaaS model requires managing tenant subscriptions, pricing tiers, and metered usage (e.g. tracking API calls, document generations, and AI token spend) while maintaining compliance isolation.

## Decision
Integrate Stripe as the commercial billing gateway. Mapped plans (`Starter`, `Professional`, `Enterprise`, `Validated Enterprise`) determine feature flag and volume constraints. The application logs usage metrics to `billing_invoices` and synchronizes them with Stripe subscriptions. We implement webhooks to handle payment renewals, failures, and tenant suspensions.

## Alternatives Considered
* **Ad-hoc Custom Billing Engine:** Rejected due to complexity, compliance overhead, and card details storage security risks (PCI-DSS).

## Risks
* **Stripe Webhook Failures:** Network delays or dropped webhooks could prevent updating local tenant states.
* **Mitigation:** Query Stripe's API on dashboard load if the local cached subscription status is within 24 hours of expiration.

## Compliance Impact
Adheres to PCI-DSS payment card data security regulations by offloading credit card storage and merchant processing to Stripe.

## Rollback Strategy
Disable external billing services, defaulting all tenants to a basic local license flag model in configuration tables.
